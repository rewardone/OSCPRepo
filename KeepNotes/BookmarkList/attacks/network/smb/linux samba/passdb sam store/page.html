On the local machine, all you have to do is
pdbedit -Lw

to automatically dump human readable information.

If you steal a passdb.tdb file, you might also have to grab the /etc/samba/smb.conf
Place passdb.tdb in your /var/lib/samba/private directory
pdbedit -Lw

options may help troubleshoot: -v and -d 10 (verbosity and debug 10)


Using the -Lw flag, passwords will be listed in the older smbpasswd format as follows:
The account information that was returned by this command in order from left to right consists of the following colon separated data:
    Login ID :  UNIX UID : Microsoft LanManager password hash (password converted to upper-case then hashed) : Microsoft NT password hash (hash of the case-preserved password) : Samba SAM Account Flags : The LCT data (password last change time).

example:
Login UNIX UID LM Hash NT Hash Acc Flag   LCT data (last change time)
