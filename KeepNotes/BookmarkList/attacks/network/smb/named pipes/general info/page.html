Named Pipes are one of the methods used to perform IPC in Windows
One-way or duplex pipe communications between pipe server and one or more pipe clients
Utilizes a unique files system called NPFS (Named Pipe Filysystem)
Any process can access named pipes, subject to security checks
All instances of a named pipe share the same pipe name, but each instance has its own buffers and handles

Many configurations
Half duplex, full duplex
Byte-oriented, packet-oriented
Local or Network!

Named pipes network communication is NOT encrypted and uses the protocols SMB (445) or DCE/RPC (135)

Named and Unnamed \ anonymous Pipes
Two types of named pipes:
• Named pipes: have a specific name, all instances share the name
• Unnamed \ anonymous pipe: is not given a name
  • o Only used for communication between a child and it’s parent process
    o Always local; they cannot be used for communication over a network
  • o Vanishes as soon as it is closed, or one of the process (parent or child) completes execution
  • o Actually named pipes with a random name


Named pipes have Access Control Lists
By default DACL (discretionary access control lists) permissions are set to everyone using anonymous login (null session)
There is a max number of instances of a pipe
