File Transfer Protocol (FTP). Zou et al [130] suggested two covert channels into the FTP. The first one
encodes covert bits directly into the FTP commands, so if there are N commands, every command will represents
log2N bits. The second one varies the number of FTP NOOP commands send during idle periods. The number
of send NOOP commands is equal to the integer value of the covert data. Before sending a new value, an ABOR
command needs to be sent. For FTP, it is normal to send NOOP or ABOR continually to prevent the control
connections from entering idle status.
