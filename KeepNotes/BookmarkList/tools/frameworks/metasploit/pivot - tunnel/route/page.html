Metasploit route command can be used to establish a route to other networks that can be
used to launch Metasploit exploits

usage: route [add|remove] <network_to_attack> <subnet> <session_to_pivot_through>

msf exploit(handler) > route add 192.168.128.0 255.255.255.0 1
[*] Route added
msf auxiliary(tcp) > route print
Active Routing Table
====================
 Subnet Netmask Gateway
 ------ ------- -------
 192.168.128.0 255.255.255.0 Session 1

There are limitations to using Meterpreter session. Only modules within
Metasploit can directly use the session. Furthermore, the sessions only transmit TCP
packets.

Note: Only use MSF route here. If you're in Meterpreter session, using 'route' will mess up the hosts routing table!

Note: The routing we provide with the run autoroute command gives us the freedom to work in the Metasploit Framework. But when we try to reach the target with a Kali tools, we need tools like port forwarding or proxychains.
