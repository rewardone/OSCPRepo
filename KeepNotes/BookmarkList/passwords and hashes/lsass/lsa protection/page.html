Windows Server 2012 R2 and Windows 8.1 and + include LSA protection which involves enabling LSASS as a protected process. (Mimikatz can bypass this with a driver, but should be noticeable in logs)

LSA Protection prevents reading memory and code injection by non-protected processes. This provides added security for the credentials that the LSA stores and manages.


Enabling LSA protection:
  • Open RegEdit and set the Key HKLM:\SYSTEM\CurrentControlSet\Control\LSA\RunAsPPL to DWORD 1
  • Create a new GPO and browse to Computer Configuration > Preferences > Windows Settings. Right click and add a new Registry Item. Add RunAsPPL under HKLM:\System\CurrentControlSet\Control\LSA
